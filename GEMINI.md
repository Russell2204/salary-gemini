# VibeFinance (salary-gemini)

Персональный финансовый трекер с интеграцией искусственного интеллекта Google Gemini для анализа расходов и доходов.

## Обзор проекта

Проект представляет собой современное веб-приложение для управления личными финансами. Пользователи могут регистрироваться, добавлять источники дохода (работы) и фиксировать расходы по категориям. Встроенный AI-консультант анализирует данные и дает советы по оптимизации бюджета.

### Технологический стек

- **Framework:** Next.js 16.2.6 (App Router) с поддержкой Turbopack.
- **Язык:** TypeScript.
- **База данных:** Prisma 7.8.0 с адаптером PostgreSQL (`@prisma/adapter-pg`).
- **Аутентификация:** Next-Auth v5 (Auth.js).
- **Искусственный интеллект:** Google Generative AI (Gemini 1.5 Flash).
- **UI/Стилизация:** Tailwind CSS 4, Lucide React (иконки), Recharts (графики), Next-themes (темная тема).

## Архитектура и структура

- `app/api/`: API-роуты для работы с расходами, работами, регистрацией и AI-анализом.
- `app/generated/prisma`: (Устарело) Ранее здесь генерировался клиент Prisma.
- `generated/client/`: Текущее местоположение сгенерированного клиента Prisma (вынесено в корень для совместимости с Turbopack).
- `components/`: Реализуемые компоненты интерфейса (дашборд, графики, формы, AI-анализ).
- `lib/prisma.ts`: Инициализация Prisma Client с использованием пула соединений `pg`.
- `prisma/schema.prisma`: Схема базы данных (модели User, Job, Expense).

## Ключевые команды

| Команда | Описание |
| :--- | :--- |
| `npm run dev` | Запуск сервера разработки |
| `npm run build` | Сборка приложения для продакшена |
| `npm run lint` | Запуск проверки кода линтером |

| `npx prisma db push` | Синхронизация схемы с базой данных |

## Особенности разработки (Prisma 7)

В проекте используется **Prisma 7**, что накладывает определенные требования:
1. Параметры подключения (`url`) не хранятся в `schema.prisma`. Они управляются через `prisma.config.ts`.
2. Клиент Prisma генерируется в `generated/client/` и импортируется в `lib/prisma.ts` через относительные пути.
3. Для работы с базой данных используется явный адаптер `PrismaPg`.

## Настройка окружения (.env)

Для полноценной работы требуются следующие переменные:
- `DATABASE_URL`: Строка подключения к PostgreSQL.
- `GEMINI_API_KEY`: Ключ API от Google AI Studio.
- `AUTH_SECRET`: Секретный ключ для сессий Next-Auth.
- `AUTH_URL`: Базовый URL приложения.
